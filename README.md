# Android 系統情報站 (System Monitor Tool) v6.0

這是一款基於 **React Native + Expo** 開發的 Android 綜合系統工具 App。旨在提供使用者一個精美、直覺且全方位的介面，即時掌握手機硬體狀態、個人健康數據以及環境資訊。

---

## 🚀 專案目的
本專案的初衷是為 Android 使用者打造一個「一站式」的監測中心。不同於傳統生硬的硬體檢測軟體，我們結合了 **Material Design 3 (MD3)** 設計語言，將冷冰冰的數據轉化為易於閱讀的視覺化圖表，並加入實用的控制工具與健康追蹤功能。

## ✨ 核心功能

### 1. 系統監測 (System Monitor)
*   **電力狀態**：即時電量百分比、充電狀態偵測。
*   **動態顏色邏輯**：電量依比例變色（紅/黃/綠），剩餘 80% 時自動發送通知提醒。
*   **資源看板**：使用 **SVG 甜甜圈圖** 顯示儲存空間與記憶體 (RAM) 使用率，剩餘空間不足 10% 時顯示紅色警告。
*   **硬體詳情**：包含裝置型號、Android 版本、核心架構、API 級別及系統運行時間 (Uptime)。

### 2. 健康追蹤 (Health Tracking)
*   **即時計步**：利用手機感應器獲取今日累積步數。
*   **目標設定**：可自定義每日步數目標，並使用 `AsyncStorage` 進行數據持久化儲存。
*   **達成提醒**：步數達成目標時，系統將觸發慶祝通知與震動反饋。

### 3. 快捷工具箱 (Control Center)
*   **手電筒控制**：一鍵開關手機閃光燈，並具備觸覺震動回饋。
*   **系統設定跳轉**：快速打開 Wi-Fi、顯示、定位等系統設定頁面。
*   **深色模式切換**：支援手動切換深色/淺色主題，提升夜間使用體驗。

### 4. 環境與網路 (Environment & Network)
*   **天氣預報**：基於地理位置自動顯示目前的氣溫與天氣狀況。
*   **網路詳情**：偵測 Wi-Fi SSID (名稱)、本機 IP 位址與連線類型。

## 🛠 技術架構
*   **開發框架**：React Native (Expo SDK 54)
*   **語言**：TypeScript / JavaScript
*   **UI 組件庫**：React Native Paper (Material Design 3)
*   **硬體存取**：
    *   `expo-battery`: 電量監測
    *   `expo-sensors`: 計步感應器
    *   `react-native-device-info`: 深層硬體數據
    *   `react-native-svg`: 視覺化圓形圖表
    *   `expo-notifications`: 本地通知系統
    *   `expo-location`: 地理定位與天氣
*   **自動化編譯**：GitHub Actions (CI/CD)

---

## 🎨 UI 畫面設計理念
*   **卡片式佈局**：將不同類型的資訊模組化，讓操作流暢且不凌亂。
*   **視覺化回饋**：使用大量的進度條與甜甜圈圖，讓使用者一掃即知系統健康度。
*   **觸覺設計**：整合 `expo-haptics`，在互動（如刷新、開關燈、切換主題）時提供不同強度的震動回饋，提升質感。

## 📦 如何打包與測試

由於本專案採用 **GitHub Actions 雲端編譯**，您無需在本地安裝 Android Studio：

1.  **推送代碼**：將專案推送到您的 GitHub 儲存庫。
2.  **觸發編譯**：前往 GitHub 專案頁面的 **Actions** 標籤，系統會自動啟動 `Build Android APK` 流程。
3.  **下載 APK**：編譯完成後（約 5-8 分鐘），在 Artifacts 區塊下載 `app-debug-apk`。
4.  **安裝測試**：將 APK 傳送到手機安裝。
    *   *註：需先在手機設定中允許「安裝未知來源應用程式」。*

---

## 📝 開發備註
*   **Wi-Fi SSID**：需要開啟 GPS 定位權限方可讀取。
*   **步數權限**：首次啟動需同意「身體活動權限」。
*   **手電筒**：基於相機 API，需同意相機存取權限。
